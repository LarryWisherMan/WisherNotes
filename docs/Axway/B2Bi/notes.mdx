## Trading and Processing

### Messasge Trading

- Partners -> Application = Inbound Flows
- Application -> Partners = Outbound Flows
- Trading can invole packaging, and encrytion of messages
- Processes involving the consumption and delivery are managed by B2Bi Trading engine

### Message Processing

- Managed by the integration engine which handles the transformation of messages between formats
- creates and sends business level acknowlegements
- processing can be applied to all consumed messages

## B2Bi Objects

### Community objects

- local way of grouping trading partners
- defines orgs internal processes for handling messages
- defines how Community expects to recive messages from applications and from remote partners
- community can be assigned the role of sender or receiver in aggreement
